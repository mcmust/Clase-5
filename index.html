Trabajando con Git

  • Lo primero que hacemos es ir a la TERMINAL o COMANDO y crear un repositorio. Vamos a la terminal, vamos con los comandos cd y ls hasta la carpeta en la que vamos a tener el erchivo con el que vamos a querer trabajar
  • git init enter para que inicie un repositorio, para que git empiece a escuchar dentro de esacarpeta.
  • Con eso le digo: git escucha lo que esta pasando en esta carpeta.
  • git status enter para que git me diga que sabe de mi codigo
  • Va a aparecer nuestro archivo index.html creado y abrierto en visual estudio code en rojo, por que aun no le dije que lo guarde o deje de ignorar
  • git add . enter nos va a poner en verde, por que git agrego y empezo a regristrar y va a trabajar sobre todos los archivos de esa carpeta
  • git add nombredelarchivox enter me va a poner en verde unicamente ese archivo y todos los que agregue de este modo -uno por uno-
  • git checkout -- nombredelacarpeta enter, git me borra los cambios hechos en esa carpeta y deja de seguirla o monitorearla
  • git commit -m "nombredelcambiooversion" enter, me guarda el cambio bajo un nombre determinado. No me lo va a volver a mostrar en status hasta que yo lo cambie
  • git log enter, me muestra los commits comandos que fui haciendo
  • Tengo que agregar los cambios (git add para pasarlos de rojo a verde) y reci√©n ahi hoy a poder hacer un commit
  • git viene escuchando los cambios que voy haciendo, guardando y generando una version a medida que creo un commit
  • Hasta aca vengo trabajando sola, al querer compartirlo con otra persona - GitHub "red social del codigo", no voy a subir el codigo solo, sino el historial del codigo para que pueda verse los pasos que fui haciendo y de permitirlo, ver que cambios hizo la otra persona
